## 2. 图解目标

用一张中密度手绘教学卡片说明 LoRA 的核心机制：基础模型原始权重保持冻结，只在关键层旁边加入可训练的低秩适配器，让模型以较低训练成本适配新任务。同时提醒读者：LoRA 不是独立小模型，也不等于完全免费的微调。

## 3. 推荐构图

- 主构图类型：C. 分层型
- 选择理由：LoRA 的核心不是线性流程，而是“基础模型主体 + 旁路低秩适配器”的结构关系。分层型可以清楚展示冻结权重、可训练适配器和任务输出之间的关系。
- 备选构图类型 1：H. 工具箱 / 组件集合型
- 不采用理由：工具箱能表达“可切换适配器”，但容易弱化 LoRA 与基础模型权重之间的低秩旁路关系。
- 备选构图类型 2：D. 对比型
- 不采用理由：LoRA 确实适合和全量微调对比，但词条首图更应先解释它怎么工作，而不是把画面主要用于 A/B 对比。
- 是否混合构图：轻微混合，在分层主画面右侧加一个小对比便签。
- 主构图边界：主画面只展示基础模型、冻结权重、低秩适配器和任务输出，不展开数学公式和具体训练超参数。
- 近期同质化检查：最近可读取的 `RAG_一图看懂brief.md` 使用 B. 流程型，常见视觉套路是横向 4 步流程卡片。本次改用分层型，以避免继续使用横向流程流水线。

## 4. 核心视觉隐喻

把 LoRA 画成一台大型机器旁边的“可拆卸微调模块”：机器主体被锁住不改动，旁边插入几块小调节片，调节片负责学习新任务。画面要保留“冻结权重”“低秩适配器”“新任务”这些真实术语，避免只剩机械类比。

## 5. 信息密度

- 密度级别：中密度
- 必画信息：基础模型、冻结原始权重、低秩适配器、只训练适配器、新任务/新风格输出。
- 应画信息：适配器可切换、训练参数更少、依附基础模型运行。
- 可删信息：rank 具体数值、矩阵公式、QLoRA 量化细节、不同框架 API。
- 禁画信息：具体厂商 Logo、未经核实的性能数字、“一定接近全量微调”“免费微调”等绝对表达。
- 如果画面过密，优先删除：QLoRA 旁注和“多任务适配器管理”说明，只保留“适配器可切换”标签。

## 6. 画面元素

- 核心形象：中间一台大模型机器或堆叠模型层，主体用锁头标记“冻结”。
- 输入 / 上游 / 背景元素：左侧小卡片写“新任务 / 新风格”，箭头指向模型。
- 输出 / 下游 / 结果元素：右侧输出卡片写“适配后回答 / 风格”。
- 场景 / 使用价值元素：模型旁边插着两块小模块，标记“低秩 A”“低秩 B”或“LoRA 适配器”。
- 边界提醒元素：右下角便签写“不是独立小模型”。
- 来自稿件的关键元素：冻结基础模型、训练少量低秩矩阵、降低训练成本、适配新任务、需要基础模型配合。
