# DPO 一图看懂 brief

## 2. 图解目标

用一张中密度手绘教学卡片说明 DPO 的核心关系：同一个问题下有“更好回答”和“较差回答”两份偏好样本，DPO 直接训练模型拉开两者概率差距，让模型更倾向生成被偏好的回答。同时提醒读者：DPO 不是没有偏好数据，也不是自动安全保证。

## 3. 推荐构图

- 主构图类型：D. 对比型
- 选择理由：DPO 最需要读者先看懂 chosen / rejected 成对偏好，以及“拉近好回答、拉远差回答”的相对关系。对比型能比流程图更直接地表达偏好差异和边界提醒。
- 备选构图类型 1：B. 流程型
- 不采用理由：流程型能展示“偏好数据 -> DPO 训练 -> 模型输出”，但容易把 DPO 画成普通训练流水线，弱化偏好对比这个核心。
- 备选构图类型 2：C. 分层型
- 不采用理由：分层型适合展示参考模型和当前模型约束，但会让初学者过早陷入训练结构，首图不必展开数学细节。
- 是否混合构图：轻微混合，在对比主画面下方加一条短训练箭头。
- 主构图边界：主画面只展示同题偏好对、DPO 训练和模型倾向变化，不画公式、不展开 PPO / IPO / KTO 等方法族。
- 近期同质化检查：最近可读取的 `LoRA_一图看懂brief.md` 使用 C. 分层型，`RAG_一图看懂brief.md` 使用 B. 流程型。本次改用 D. 对比型，避免继续使用分层或横向流程套路。

## 4. 核心视觉隐喻

把 DPO 画成“老师对比两份作文后给模型调方向”：左边是同一个问题，中央是两张答案卡，一张被盖上绿色“更好”，一张被盖上灰色“少选”。DPO 像一只调节旋钮，把模型输出偏向更好回答，但旁边保留“仍需好数据评估”的提醒。

## 5. 信息密度

- 密度级别：中密度
- 必画信息：同一个问题、偏好回答、不偏好回答、DPO 训练、模型更偏向好回答、参考模型约束。
- 应画信息：不用单独奖励模型、偏好数据质量关键、训练后仍需评估。
- 可删信息：DPO 损失公式、β 超参数、具体 TRL 参数、PPO 细节、各种 DPO 变体。
- 禁画信息：具体厂商 Logo、性能数字、“自动安全”“完全替代 RLHF”“不需要人类偏好”等绝对表达。
- 如果画面过密，优先删除：参考模型约束的小注释，其次删除“少一步奖励模型”标签，但保留偏好数据质量提醒。

## 6. 画面元素

- 核心形象：中间两张并排回答卡片，绿色卡片标“更好回答”，灰色卡片标“较差回答”。
- 输入 / 上游 / 背景元素：左侧同一个问题气泡，箭头指向两张回答卡。
- 输出 / 下游 / 结果元素：右侧模型小面板，箭头更粗地指向“偏向好回答”。
- 场景 / 使用价值元素：中央有“DPO 训练”调节旋钮或小训练台，旁边便签写“少一步奖励模型”。
- 边界提醒元素：右下角黄色便签写“仍需好数据评估”。
- 来自稿件的关键元素：偏好数据、chosen / rejected、直接优化、参考模型约束、不训练显式奖励模型、不是安全保证。
