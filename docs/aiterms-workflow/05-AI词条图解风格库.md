# AI 词条图解风格库

本文档是 `05-AI词条一图看懂_prompt.md` 的风格详细库。

使用规则：

- 本文档只在 `05-AI词条一图看懂_prompt.md` 已经选定主视觉风格后使用。
- 每次只读取与已选风格编号匹配的一个条目。
- 不要通读全库，不要读取未选风格，不要引用或混合其他风格条目。
- 生成图片提示词时，只复制已选风格条目的要点，不要把其他风格条目混入。
- 风格只约束视觉语言，不决定信息结构；画面结构仍以已选构图为准。
- 如果风格条目中的可用元素与已选构图冲突，以构图为准；不要因为风格里提到卡片、桌面、工具台等元素，就改变主构图。
- 所有风格都必须保持 16:9 横版中文信息图、文字清晰可读、中密度、适合长期知识库网站。

## S1 手绘教学卡

- 16:9 横版中文手绘教学卡片。
- 米白色纸张背景。
- 马克笔 / 签字笔线条。
- 轻微手绘感，但清晰工整。
- 细线圆角边框。
- 柔和色块：浅绿、浅紫、浅黄、浅蓝、浅灰。
- 可用小图标、拟人小助手、工具、卡片、线缆、箭头、便签、气泡。
- 可爱但克制，适合长期知识库网站。
- 更像轻松入门的知识卡片，适合抽象概念和基础解释。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要真实照片、复杂 3D、赛博朋克、霓虹、夸张科技感、营销海报感。
- 不要密集小字，不要假 UI 截图，不要伪造品牌 Logo。

## S2 桌面资料图

- 16:9 横版中文桌面资料信息图。
- 浅色桌面或纸面背景，干净低噪声。
- 使用文件夹、标签、便签、资料卡、引用标记、连接线、轻量图标。
- 画面像把一组资料整理在桌面上，而不是广告海报。
- 适合标准、框架、文档、内容结构、知识库、引用来源、资料流转。
- 色彩克制，常用浅蓝、浅绿、米白、浅灰，少量橙色提醒。
- 可以有层叠文件、资料夹、卡片关系和小型看板，但不要喧宾夺主。
- 和 S10 的区别：S3 侧重资料整理、来源、引用和内容结构，不侧重工具或系统功能拆解。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要真实照片、复杂 3D、杂乱办公桌、营销增长漏斗。
- 不要伪造真实产品界面、品牌 Logo、来源编号或未经确认的数据。

## S3 架构蓝图

- 16:9 横版中文架构蓝图信息图。
- 深浅分层背景，可以是浅蓝灰或深蓝灰，但必须保证文字清晰。
- 使用节点、接口、模块、协议标签、连接线、分层框和数据流箭头。
- 画面要像清楚的系统结构图，而不是炫技科技海报。
- 适合基础设施、协议、运行时、中间层、模型服务、系统栈和互操作。
- 可用模块边框、端口、小型网格、层级标题、接口标签。
- 色彩以蓝、青、灰为主，少量绿色或橙色强调关键路径。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要赛博朋克、霓虹发光、复杂 3D 服务器、真实云厂商 Logo。
- 不要把所有元素画成密集网络，避免节点太多导致不可读。

## S4 场景隐喻

- 16:9 横版中文场景隐喻信息图。
- 使用现实物件或简化场景做隐喻，但不使用真实照片。
- 场景必须服务概念理解，不能只是气氛插画。
- 每个场景物件都必须能对应词条中的一个具体对象、机制或边界；不能只做氛围背景。
- 画面可以有桌面、道路、迷宫、工坊、传送带、舞台、地图、控制台等隐喻物件。
- 适合难解释的抽象概念、社区词、风险边界、能力边界和读者容易误解的主题。
- 图中文字应更克制，用少量标签把隐喻对象和概念关系对应起来。
- 色彩要柔和、低刺激，避免戏剧化光影。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要真实照片、电影海报感、过度拟人、夸张表情或儿童绘本化。
- 不要为了隐喻牺牲准确性；隐喻对象必须能对应当前词条的专属机制。

## S5 对比海报

- 16:9 横版中文对比澄清信息图。
- 可使用明确分区和对照标记，但对比结构由已选构图决定。
- 使用明确分区、对照标签、少量强调色、勾叉、警示标记和纠偏说明。
- 适合误区澄清、相近概念对比、错误理解 vs 正确定位。
- 视觉应像清楚的编辑型解释海报，不像营销宣传图。
- 色彩建议一侧用冷色表示真实定位，另一侧用暖色表示误区或风险。
- 文字层级要清楚，避免每栏塞满长句。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要做成夸张辩论海报，不要使用煽动性标题。
- 不要把复杂概念简化成绝对对错；避免虚构数据、排名或权威背书。

## S6 极简知识地图

- 16:9 横版中文极简知识地图。
- 浅色背景，大量留白，线框模块和细连接线。
- 使用节点、区域、路径、位置标记、轻量图标和少量文字标签。
- 适合生态位置、上下游关系、知识图谱、对象分布、搜索路径或多概念关系。
- 画面要像一张清楚的知识地图，而不是完整报告。
- 色彩以蓝灰、浅绿、浅橙为主，强调色只用于主路径或关键节点。
- 模块数量要克制，避免密集网状图。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要画成复杂地铁图、社交网络图或满屏节点。
- 不要让装饰地图元素压过概念关系。


## S7 期刊图解风

- 16:9 横版中文期刊图解信息图。
- 白底或极浅灰背景，细线结构图，编号标注，少量强调色。
- 画面像论文配图或技术期刊插图，但更亲和、更适合普通读者。
- 适合算法、评估、标准、模型结构、指标、训练机制和严肃技术概念。
- 可用编号、细箭头、结构部件、简化变量、模块标签和边界说明。
- 色彩以蓝、灰、绿色为主，少量橙色标出风险或关键步骤。
- 保持严谨、干净、低装饰，不要为了好看牺牲结构准确。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要满屏公式、论文截图、过密脚注、真实图表截图。
- 不要伪造实验结果、参数、排行榜或论文来源。
