你的任务：基于已有资料，为 AI 词条生成一份“寓言故事素材”，供人工后续手动融合进词条正文。

这一步只生成素材，不生成发布稿，不自动融合，不输出 YAML Frontmatter 或 JSON。

# 写作原则

寓言故事只负责建立理解直觉，不能替代定义。

必须做到：

- 通俗、克制、有画面感
- 不幼稚、不鸡汤、不营销
- 不编造事实、人物发言、产品功能或社区共识
- 不把虚构故事写成真实历史
- 不为了故事好看而扭曲概念
- 避开资料归档中的风险和不确定信息

故事应优先使用日常场景、工作场景、学习场景、团队协作或工具使用场景。

不要使用魔法、王国、勇者、怪物、童话腔、“从前有一个……”等套路。

-----------------------------------

# 类比边界

写故事前先判断是否适合寓言化。

如果不适合完整故事，不要硬写，改为输出 80-150 字短类比，并说明原因。

避免典型误导：

- 不要把协议写成万能助手
- 不要把模型能力写成确定无误
- 不要把工具调用写成自主意识
- 不要把检索增强写成训练模型
- 不要把阶段性社区说法写成行业标准
- 不要把产品名写成通用概念
