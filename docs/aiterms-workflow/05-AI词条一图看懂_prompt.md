# 内容目标

这张图不是单纯的概念入口图，也不是高密度资料海报，而是介于二者之间的「中密度资料小总结」：

- 比一句类比更完整，读者看完后应能大致说清这个词条涉及哪些关键东西。
- 比完整资料总结更克制，不把正文、参考资料和所有细节都塞进图里。
- 优先呈现“理解这个词必须知道的结构、关系、流程、边界”，而不是追求视觉装饰。

它应该让读者快速知道：

- 这个概念本质上是什么。
- 它解决什么问题，为什么会被提出来。
- 它由哪些关键部分、角色、步骤或关系构成。
- 它和模型、用户、数据、工具、系统、流程或生态之间是什么关系。
- 它常见于哪些场景，或通常怎样被使用。
- 它最容易被误解的边界是什么。

# 核心关系判断

判断这张图最该解释哪种关系：

- 连接关系：谁连接谁、统一什么接口、边界在哪里。
- 流程关系：输入如何一步步变成输出。
- 分层关系：系统层、模型层、能力层或运行栈如何上下分工。
- 工具集合：一组能力如何组合成工作台或平台。
- 对比纠偏：它不是什么，更准确是什么。
- 地图总览：多个对象在生态、风险或知识图中的相对位置。
- 其他关系：不在上述的关系。

核心关系决定构图；视觉风格只是在这个关系上选择合适的表现方式。

# 风格

- 16:9 横版中文手绘教学卡片。
- 米白色纸张背景。
- 马克笔 / 签字笔线条。
- 轻微手绘感，但清晰工整。
- 细线圆角边框。
- 柔和色块：浅绿、浅紫、浅黄、浅蓝、浅灰。
- 可用小图标、拟人小助手、工具、卡片、线缆、箭头、便签、气泡。
- 可爱但克制，适合长期知识库网站。
- 更像轻松入门的知识卡片，适合抽象概念和基础解释。
- 风格只约束视觉语言，不决定信息结构；构图仍以已选构图为准。
- 不要真实照片、复杂 3D、赛博朋克、霓虹、夸张科技感、营销海报感。
- 不要密集小字，不要假 UI 截图，不要伪造品牌 Logo。

# 信息密度规则

默认使用「中密度」。

一张图必须覆盖 5 类信息：

1. 定义锚点：一句话说明它是什么。
2. 关键组成：列出 3-8 个必须知道的组件、角色、步骤或相关对象。
3. 关系结构：说明这些东西之间怎么连接、流动、分层、对比或协作等等。
4. 使用/价值：说明它通常解决什么问题，或在哪类场景中出现。
5. 边界提醒：指出一个最容易误解的点。

这 5 类信息是内容层级，不是固定版式。不要强制做成五宫格，也不要为了凑满五类信息牺牲画面可读性。

文字总量建议控制在 60-140 个中文字符左右；必须分层清楚，布局合理，避免所有图片都变成高密度课件。

如果素材太多，按下面顺序删减：

1. 删掉重复解释正文已有细节的句子。
2. 删掉非核心场景。
3. 合并相近组件。
4. 保留边界提醒，不要为了美观删掉最关键的误解点。

如果素材太少，优先补足：

1. 它连接/影响的对象。
2. 它内部的关键组成或流程。
3. 它和相近概念的边界。
